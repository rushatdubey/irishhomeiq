Why IrishHomeIQ exists

Built to make Irish property data legible.

280k+ transactions 26 counties PSRA verified 12-stage pipeline CSO wage data

Ireland's housing market produces massive volumes of pricing and transaction data, yet most analysis remains fragmented, delayed, or oversimplified. IrishHomeIQ transforms real Irish property data into a structured analytics platform for understanding affordability deterioration, market momentum, mortgage stress, supply shortages, and regional investment opportunity.

Every figure is traceable to the official Property Price Register (PSRA). The platform is built for mortgage lenders, property funds, pricing analysts, and anyone who needs to understand the forces reshaping the Irish housing market beyond the headline price number.

SQL highlight: Price momentum with rolling windows
PostgreSQL · window functions · production-grade analytics
WITH monthly AS (
  SELECT county, DATE_TRUNC('month', date_of_sale) AS month,
    PERCENTILE_CONT(0.5) WITHIN GROUP (ORDER BY price_eur) AS median_price
  FROM transactions GROUP BY county, month
)
SELECT county, month, median_price,
  ROUND(AVG(median_price) OVER (
    PARTITION BY county ORDER BY month
    ROWS BETWEEN 5 PRECEDING AND CURRENT ROW
  ), 0) AS rolling_6m_avg,
  ROUND((median_price / LAG(median_price, 3) OVER (
    PARTITION BY county ORDER BY month) - 1) * 100, 1) AS momentum_3m_pct
FROM monthly ORDER BY county, month;

SQL highlight: Sub-county postcode analysis
D01–D24 granularity · nobody else does this for the Irish market
SELECT
  REGEXP_REPLACE(eircode, '(D\d{1,2}W?).*', '\1') AS postcode,
  CASE WHEN postcode IN ('D01','D02','D04','D06','D6W','D08') THEN 'Inner City'
       WHEN postcode IN ('D09','D11','D12','D14','D16')          THEN 'Mid City'
       ELSE 'Outer City' END AS zone,
  PERCENTILE_CONT(0.5) WITHIN GROUP (ORDER BY price_eur) AS median_price,
  COUNT(*) AS transactions
FROM transactions
WHERE county = 'Dublin' AND year = 2024
GROUP BY postcode, zone HAVING COUNT(*) >= 20
ORDER BY median_price DESC;

SQL highlight: FTB mortgage affordability gap
Irish macro-prudential limits: LTI 4×, LTV 90%, stress rate 6%
SELECT county, year, median_price,
  avg_annual_wage * 4                          AS max_mortgage_lti,
  avg_annual_wage * 4 / 0.90                   AS max_affordable_price,
  median_price - (avg_annual_wage * 4 / 0.90)  AS affordability_gap_eur,
  ROUND((median_price - avg_annual_wage*4/0.90)
        / median_price * 100, 1)               AS gap_pct,
  CASE WHEN gap_pct > 20 THEN 'Cannot Afford: Major Gap'
       WHEN gap_pct > 5  THEN 'Stretched: Borderline'
       ELSE                   'Within LTI Limit' END AS ftb_status
FROM affordability WHERE year = 2024 ORDER BY gap_pct DESC;

SQL highlight: Risk-adjusted return ranking
Growth + yield model, penalising affordability stress
SELECT county, region, median_price_2024,
  price_growth_4yr_pct, rental_yield_est_pct, years_salary_to_buy,
  ROUND(
    (price_growth_4yr_pct * 0.4 +
     rental_yield_est_pct * 10 * 0.4 -
     years_salary_to_buy  * 2  * 0.2), 1
  ) AS risk_adj_return_score,
  RANK() OVER (ORDER BY risk_adj_return_score DESC) AS risk_adj_rank
FROM investment_scores
ORDER BY risk_adj_return_score DESC;
